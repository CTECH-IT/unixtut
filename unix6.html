UNIX Tutorial Six

Other useful UNIX commands  

df

The df command reports on the space left on the file system. For example, to find out how much space is left on the fileserver, type

% df .

du

The du command outputs the number of kilobyes used by each subdirectory. Useful if you have gone over quota and you want to find out which directory has the most files. In your home-directory, type

% du /home/ec2-user/environment

gzip

gzip reduces the size of a file, thus freeing valuable disk space. For example, type

% cd /home/ec2-user/environment/unixstuff

% ls -l words

if you don't see the words file there, make sure you have copied it as specified in tutorial 2

Note the size of the wordsfile. Then to compress it, type

% gzip words

This will compress the file and place it in a file called words.gz

To see the change in size, type ls -l again.

To uncomress the file, use the gunzip command.

% gunzip words.gz

file

file classifies the named files according to the type of data they contain, for example ascii (text), pictures, compressed data, etc.. To report on all files in your home directory, type

% file *

find

find helps you locate a file somewhere in the folder structure. It is very powerful and can take many different arguments (try "man find" to see them all). Usually, you give it the place you want to start searching and the name you are looking for:

% find . -name somefile

Remember, you can also use "wildcards" like * and ?

% find . -name some*

% find . -name s?me?ile

An example for us would be trying to remember where you put README.bak

% cd /home/ec2-user/environment/

% find . -name README.bak

Where did it find it? If it didn't, ask Mr. B for help

history

The bash shell keeps an ordered list of all the commands that you have entered. Each command is given a number according to the order it was entered.

% history (show command history list)

Since we are using the bash shell, we can use the exclamation character (!) to recall commands easily.

% !! (recall last command)

% !-3 (recall third most recent command)

% !5 (recall 5th command in list)

% !grep (recall last command starting with grep)

You can increase the size of the history buffer by typing

% HISTSIZE=1000
